Not a template, more like an operating cycle.

How do you bring AI into development cycle without losing technical control, and without turning codebase into a slot machine?

What is this?

A local independent CLI, Specs, Context, and AgentSkills to produce different applications with more repeatable output quality.

Open by it's core: the value is not in reusing UI code, but in reusing how context is transferred to the agent.

CLI / Spec / Context / Agents

Frameworkless frontend with AI agents, quality gates, and repeatable delivery from web to Electron.

Vibe Coding with Rigor

Not vibe coding that goes off the rails. Vibe coding with discipline, verification, and repeatable output.

The operating pipeline

  1. CLI — collects domain, users, scope, rules, edge cases
  2. SPEC — turns input into decisions, acceptance criteria, schema
  3. CONTEXT — becomes local project memory
  4. AGENTS — instructions, prompts, skills, style guide
  5. OUTPUT — different apps, same quality cycle
