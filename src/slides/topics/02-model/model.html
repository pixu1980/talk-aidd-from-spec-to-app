The real pipeline

From domain to output: every step is traceable, recoverable, and repeatable.

CLI → Spec → Context → Agents → Output

  1. CLI — collects domain, users, scope, rules, edge cases
  2. SPEC — turns input into decisions, acceptance criteria, schema
  3. CONTEXT — becomes local project memory
  4. AGENTS — instructions, prompts, skills, style guide
  5. OUTPUT — different apps, same quality cycle
