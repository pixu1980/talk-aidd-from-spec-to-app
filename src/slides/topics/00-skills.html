What are Agent Skills?

Lightweight, open format for extending AI agent capabilities with specialized knowledge and workflows. Based on agentskills.io: open standard, portable skills, on demand loading.

Skills = procedures, not snippets

Do not rely only on model memory or current chat state.

Skill folder anatomy

Core idea: a folder with a SKILL.md file. The file defines metadata and instructions, then can carry support files.
  • SKILL.md: required name, description, instructions
  • scripts/: optional executable code
  • references/: optional docs and playbooks
  • assets/: optional templates and resources

How skills work

  1. Discovery: agent loads only name and description first
  2. Activation: matching task loads full SKILL.md into context
  3. Execution: agent follows instructions, optional code, optional files

Why skills matter

A good skill says when to load, how to work, what to avoid, and how to verify.

Reduces variance

  • Do not depend on model memory only
  • Do not depend on current chat state only
  • Procedure stays repeatable and auditable

Increases portability

  • Same procedure works across projects
  • Domain expertise encoded once, reused many times
  • Open and portable, not black box behavior
Point to the three stage model: discovery, activation, execution. Tie it to versioned folders and portable procedures.
